184  * Struct_PerformanceMonitoring

186  * @description: An important aspect of creating estimates over future performances is to be able to aggregate prior
187  * performances of threads. The possibilities of how to aggregate such performance may vary depending on the application itself.

189  * This structure is planned for 'per-thread' use, since it will hold aggregate information over prior performances of a thread.
190  *

195  /*
196  * We assume here that there are two kinds of resources (i.e., main resources and child resources)
197  * For example, a NUMA node may refer as the main resource, and the underlying CPU may refer to as a child resource.
198  * For now, it is assumed that there is a 'single' child resource for each main resource, however the current setup
199  * may as well be expanded to the case of multiple child resources.
200  */
201  std::string resource_; // This is the type of resource over which these measurements are recorded.
202  unsigned int num_sources_; // This is the number of choices for the main resource
203 
204  /*
205  * Current Performances
206  */
207  double performance_; // Current performance of the thread w.r.t. the resource at hand.
208  double balanced_performance_; // A special type of performance that takes into account the variance from the mean (over all other threads).
209  double alg_performance_; // A special type of performance (to be defined).
210 
211  bool performance_update_ind_; // Indicates whether the performance has been updated
212 
213  /*
214  * Overall performance indicators (over all threads)
215  */
216  double overall_performance_;
217  double overall_balanced_performance_;

220  /*
221  * Statistical Properties
222  */
223  double min_performance_; // minimum performance observed so far
224  double max_performance_; // maximum performance observed so far

226  /*
227  * Running Average Performances Per Main Resource
228  */
229  double run_average_performance_; // running average performance over the main resource
230  double run_average_balanced_performance_; // running average balanced performance over the main resource

232  double run_average_balanced_performance_before_;
